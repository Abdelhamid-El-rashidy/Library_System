MUST Be Micro Commits: Each commit should represent one logical change. Avoid bundling multiple changes into one commit. 

Key Commit Typesfeat: A new feature.fix: A bug fix.docs: Changes to documentation.style: Formatting, missing semicolons, etc. (no code changes).refactor: Code restructuring that neither fixes a bug nor adds a feature.perf: Performance improvements.test: Adding or correcting tests.chore: Updating tasks, build system, or package manager (e.g., config).Best PracticesUse Imperative Mood: "Add feature" not "Added feature".Be Concise: Keep the first line under 50-72 characters.Capitalize: Capitalize the subject line.No Period: Do not end the subject line with a period.Breaking Changes: Use ! after the type (e.g., feat!: change API).Body Content: Use the body to explain what and why, not how.
